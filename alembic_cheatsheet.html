Workflow du quotidien

modifier models/ → revision --autogenerate → relire le fichier → upgrade head

Créer une migration

créer
docker compose exec api alembic revision --autogenerate -m "description" Génère automatiquement un fichier dans versions/ en comparant les modèles à la DB
créer
docker compose exec api alembic revision -m "description" Crée un fichier vide — à remplir manuellement (sans autogenerate)

Appliquer des migrations

appliquer
docker compose exec api alembic upgrade head Applique toutes les migrations en attente jusqu'à la dernière
appliquer
docker compose exec api alembic upgrade +1 Applique uniquement la prochaine migration
appliquer
docker compose exec api alembic upgrade a1b2c3d4 Applique jusqu'à une révision précise (ID du fichier versions/)

Annuler des migrations

annuler
docker compose exec api alembic downgrade -1 Annule la dernière migration appliquée
annuler
docker compose exec api alembic downgrade base Annule toutes les migrations — remet la DB à zéro

Inspecter l'état

info
docker compose exec api alembic current Affiche la version actuellement appliquée en base
info
docker compose exec api alembic history Liste toutes les migrations dans l'ordre chronologique
info
docker compose exec api alembic history --verbose Idem avec le détail de chaque révision
info
docker compose exec api alembic heads Affiche la ou les dernières révisions disponibles

Sans Docker (venv local)

local
alembic upgrade head Même commandes, sans le préfixe docker compose exec api
